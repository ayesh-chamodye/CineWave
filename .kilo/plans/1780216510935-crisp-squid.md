# Plan: Add Download Initiation to Downloads Page + Stream Endpoint for TV

## Goal
Let users search/browse and start new downloads directly from the Downloads tab. TV downloads include season/episode selection. Reuse the existing `/api/stream` endpoint already wired in the download bloc.

---

## Scope Assertion
- Existing `StreamRepository` → `StreamRemoteDataSource` → `ApiEndpoints.stream()` already supports `tmdbId`, `type`, optional `season`/`episode` via `/api/stream?id=...&type=...`.
- `tv_detail_page` already has season/episode UI.
- `SourceSelectionDialog` already handles `VylaSource` quality list.
- Gap: Downloads page has no way to initiate new downloads; user must navigate to detail pages to find the download button.

---

## Files to Change

### 1. `lib/features/downloads/presentation/pages/downloads_page.dart`
**Replace** (or extend) the Downloads page to include:
- A search row at the top (TextField + search action button)
- `BlocListener<SearchBloc, SearchState>` to watch for search results, separated from the existing `BlocBuilder<DownloadBloc, ...>`
- A `_SearchResultsSection` list showing matched movies & TV shows
- On result tap:
  - For **movie**: open the existing `SourceSelectionDialog`, then on source chosen emit `StartDownload(movie: ..., url: source.m3u8)`
  - For **TV show**: first show a simple `SeasonEpisodePickerDialog` (new widget, small), then show `SourceSelectionDialog`, then emit `StartDownload(tvShow: ..., season: s, episode: e, url: source.m3u8)`
- Keep existing download list below; when no downloads and no search, show “No downloads yet”, otherwise show “Search to add new downloads” prompt when search section is empty

### 2. `lib/features/downloads/presentation/widgets/season_episode_picker_dialog.dart` (new)
- Simple `AlertDialog` with a season dropdown and episode grid (5 columns), mirroring `_buildSeasonDropdown` / `_buildEpisodeGrid` from `tv_detail_page.dart:137-207`
- Parameters: `TVShow tvShow`, required `int Function(int season, int episode) onConfirm`

### 3. `lib/features/downloads/presentation/widgets/search_result_tile.dart` (new)
- Tile widget for search results
- Shows poster, title, type badge (“Movie” / “TV Show”)
- OnTap opens SeasonEpisodePicker (if TV) or SourceSelection dialog (if movie)
- Reuses `NetworkImageWidget`

### 4. `lib/features/search/presentation/widgets/search_result_list.dart`
**Extend** the widget to accept an `onTap` callback so the same search result tile styling works in both Search and Downloads features. Alternatively, create a new shared `MediaResultTile` widget in `shared/widgets/` and consume it from both pages.

### 5. `lib/shared/widgets/media_result_tile.dart` (new)
- Extracted result tile used by `SearchResultList` and by the downloads page search results
- Parameters: `MediaItem` (new unified model, or accept either `Movie`/`TVShow` directly — accepting both via factory is fine)

---

## Data Flow Summary
1. User taps search field on Downloads page  
2. User enters query → `SearchBloc.add(SearchMovies(query))` + `SearchBloc.add(SearchTvShows(query))`
3. Results appear in embedded section
4. User taps a TV result → `SeasonEpisodePickerDialog` confirms season/episode → `SourceSelectionDialog` calls `/api/stream?id=tmdbId&type=tv&season=&episode=` (bytes fetched in `StreamRemoteDataSource`) → user picks source → `DownloadBloc.add(StartDownload(tvShow:..., season:..., episode:..., url: source.m3u8))`
5. User taps a Movie result → `SourceSelectionDialog` calls `/api/stream?id=tmdbId&type=movie` → user picks source → `DownloadBloc.add(StartDownload(movie:..., url: source.m3u8))`
6. New download item appears in the existing list via `LoadDownloads`

---

## Tests / Validation
- Manual: verify search in Downloads tab returns results and download starts for movie and TV (with season/episode symbols) in list
- Verify existing detail-page download flow still works (no regression)

## Tradeoffs / Questions
- Unifying search result tile may add a small abstraction; acceptable because both pages share the same UI.
- Season/episode picker duplicated from `tv_detail_page`; acceptable as a small reusable widget, keeping `tv_detail_page` unchanged.
